# セキュリティ監査レポート

## 1. SQLインジェクション

### 評価: **安全**

- **現状**: 全DBアクセスで Supabase JS クライアントを使用
- **根拠**: `.from()`, `.select()`, `.eq()`, `.insert()`, `.update()`, `.delete()` は全てパラメータ化されたクエリを生成
- **確認箇所**:
  - `koifes-db.ts`: ユーザー入力はオブジェクトとして渡され、文字列連結なし
  - ログインコード: dev-login はコードで `.eq()` で単一レコード取得（生SQL不使用）
  - QRコード解析: `/api/lookup-by-code` でコードからユーザー取得

**対応不要**

---

## 2. XSS（クロスサイトスクリプティング）

### 評価: **安全**

- **現状**: `dangerouslySetInnerHTML` は未使用
- **根拠**: React は `{variable}` を自動エスケープ
- **確認箇所**:
  - ニックネーム、趣味、自己紹介などユーザー入力は `{p.nickname}` 等で表示
  - 外部取得データの直接DOM挿入なし

**対応不要**

---

## 3. CSRF（クロスサイトリクエストフォージェリ）

### 評価: **要注意**

- **現状**: `/api/admin-auth` の POST に Origin/Referer チェックなし
- **リスク**: 悪意あるサイトから管理者ログインパスワード送信の誘導が可能（パスワード推測が必要なため影響は限定的）
- **対策**: Origin ヘッダーの検証を追加

---

## 4. ユーザー情報APIの制限（対応済み）

### 評価: **対応済み**

- **修正内容**:
  1. `load()` を `/api/app-data` 経由に変更。認証必須、他ユーザーは最小限（id, nickname, age, job）のみ返す
  2. 一覧取得: 接続・評価に関わるユーザーのみ、ニックネーム等の最小限の情報
  3. 詳細プロフィール: `/api/user-profile/:id` で本人または接続者のみ取得可能
  4. 管理者用: `/api/admin-data` は ADMIN_EMAILS 認証 + SUPABASE_SERVICE_ROLE_KEY で全データ取得
  5. RLS 強化: `koifes_users` の SELECT を本人の行のみに制限（013 マイグレーション）
